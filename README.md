# Book Fast Track TT

https://chabanovx.github.io/tiktok-book/ - веб версия

`Book Fast Track TT` — Flutter-приложение для быстрого чтения книг в формате RSVP (Rapid Serial Visual Presentation). Вместо обычной страницы приложение показывает слова последовательно, по одному за раз, с управляемой скоростью и акцентом на начале слова в стиле bionic reading. Это снижает количество лишних движений глазами и помогает сфокусироваться на тексте.

Проект работает офлайн, хранит библиотеку локально, запоминает прогресс чтения и позволяет быстро возвращаться к книге с того места, где пользователь остановился.

## Какую проблему решает приложение

Обычное чтение длинного текста на экране перегружает внимание:

- взгляд постоянно перемещается по строкам;
- скорость чтения сильно зависит от вёрстки, размера текста и отвлекающих элементов;
- при возврате к книге легко потерять место;
- длинные PDF и TXT неудобно читать на маленьком экране.

`Book Fast Track TT` решает эту задачу так:

- импортирует книгу из `PDF` или `TXT`;
- нормализует и подготавливает текст для показа;
- отображает слова в RSVP-режиме;
- позволяет менять скорость чтения;
- сохраняет прогресс и библиотеку локально;
- даёт полнотекстовый режим для быстрого перехода к нужному месту;
- поддерживает русский и английский интерфейс.

## Основные возможности

- Импорт книг из файлов `PDF` и `TXT`.
- Локальная библиотека с сохранёнными книгами.
- Выбор книги, запуск чтения и повторное открытие уже импортированного контента.
- RSVP-экран чтения с отображением слов по одному.
- Bionic-разметка слова: начало слова выделяется сильнее для более быстрого визуального распознавания.
- Управление скоростью чтения в `WPM` (words per minute).
- Пауза, продолжение и выход из чтения.
- Прогресс-бар и счётчик уже прочитанных слов.
- Автосохранение прогресса чтения.
- Полнотекстовый экран со списком токенов и переходом к нужной позиции.
- Удаление книги из библиотеки.
- Локализация интерфейса: `English` и `Русский`.
- Переключение темы: `System`, `Light`, `Dark`.
- Отдельный `Widgetbook` для изолированной разработки UI-компонентов.

## Как устроен проект

Приложение построено вокруг нескольких ключевых сценариев:

1. Пользователь импортирует книгу через системный файловый пикер.
2. Книга парсится в зависимости от формата (`PDF` или `TXT`).
3. Текст очищается, нормализуется и разбивается на слова.
4. Слова кэшируются локально в JSON-файл, а метаданные книги и прогресс сохраняются в SQLite через `Drift`.
5. При чтении используется RSVP-движок, который показывает токены с заданной скоростью.
6. Прогресс чтения периодически синхронизируется с локальной БД.

## Поддерживаемые функции по модулям

### Библиотека

- отображение сохранённых книг;
- пустое состояние и состояние загрузки;
- выбор активной книги;
- удаление книги;
- восстановление библиотеки при следующем запуске приложения.

### Импорт и подготовка текста

- выбор файла через `file_picker`;
- поддержка `PDF` и `TXT`;
- очистка текста от лишних переносов и части пунктуации;
- преобразование текста в список слов для дальнейшего RSVP-показа.

### Экран чтения

- запуск с последней сохранённой позиции;
- динамическое отображение текущего слова;
- регулировка скорости чтения;

- отображение прогресса и количества прочитанных слов;
- завершение книги с диалогом об окончании чтения.

### Полнотекстовый режим

- просмотр всего текста как списка токенов;
- автоскролл к текущей позиции;
- переход к выбранному слову по тапу.

- тема приложения: системная, светлая, тёмная;

## Хранение данных

Проект использует полностью локальную модель хранения:

- метаданные книг и прогресс чтения сохраняются в SQLite через `Drift`;
- подготовленный текст книги кэшируется в JSON-файлы внутри директории приложения;
- выбранная тема и язык сохраняются через `SharedPreferences`.

Это означает, что приложению не нужен сервер для базового сценария работы.

## Архитектурные заметки

- В проекте используется разделение на `data`, `domain` и `presentation` там, где это имеет смысл для фичи.
- Навигация централизована через `NavigationService` и `AppRouter`.
- Состояние библиотеки и чтения реализовано через `BLoC`.
- Локализация подключена через ARB-файлы и `flutter gen-l10n`.
- UI-компоненты вынесены в `ui_kit`, а их изолированная отладка ведётся через `Widgetbook`.
